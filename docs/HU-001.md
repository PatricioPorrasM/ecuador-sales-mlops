## Historia de Usuario

**Como** analista de datos o tomador de decisiones de negocio,  
**quiero** ingresar una pregunta en lenguaje natural sobre ventas provinciales del Ecuador a través de una interfaz de chat web,  
**para** obtener una predicción comprensible sin necesidad de conocer los detalles técnicos del modelo ni construir peticiones API manualmente.

---

## Descripción

El sistema debe ofrecer una interfaz conversacional accesible desde el navegador, donde el usuario pueda escribir preguntas como *"¿Cuánto venderán las sociedades de Guayas en julio?"* y recibir una respuesta en lenguaje natural que incluya la predicción generada por el modelo de machine learning, contexto interpretativo y los datos técnicos de respaldo de forma colapsable.

---

## Criterios de Aceptación

### Escenario 1: Pregunta válida con provincia y mes reconocibles

**Given** que el usuario está en la interfaz de chat  
**And** escribe una pregunta que menciona una provincia válida del Ecuador y un mes del año  
**When** presiona el botón de enviar o la tecla Enter  
**Then** el sistema muestra un indicador visual de "procesando"  
**And** en menos de 10 segundos aparece una respuesta en lenguaje natural con el valor de predicción  
**And** la respuesta incluye la provincia identificada, el mes identificado y el valor predicho formateado en dólares  
**And** se muestra un panel colapsable con los datos técnicos: modelo utilizado, versión, score de confianza y payload enviado al modelo  

### Escenario 2: El historial de conversación se mantiene en sesión

**Given** que el usuario ha realizado al menos una consulta previa en la sesión actual  
**When** realiza una nueva consulta  
**Then** los mensajes anteriores permanecen visibles en el chat  
**And** los nuevos mensajes se agregan al final del historial  
**And** el scroll se posiciona automáticamente en el mensaje más reciente  

### Escenario 3: Pregunta ambigua o con datos insuficientes

**Given** que el usuario escribe una pregunta que no contiene provincia o mes identificable  
**When** el sistema procesa la pregunta  
**Then** el agente responde solicitando la información faltante de forma amigable  
**And** no se muestra un error técnico ni un stack trace  

### Escenario 4: Accesibilidad del servicio

**Given** que el servicio web-ui está desplegado y corriendo  
**When** el usuario accede a la URL raíz del servicio  
**Then** la página carga completamente en menos de 3 segundos  
**And** el campo de input recibe el foco automáticamente  
**And** la interfaz es funcional en los navegadores Chrome, Firefox y Edge en sus versiones actuales
