## Historia de Usuario

**Como** analista de datos o tomador de decisiones de negocio,
**quiero** ingresar una pregunta en lenguaje natural sobre ventas provinciales del Ecuador especificando provincia, mes **y año** a través de una interfaz de chat web,
**para** obtener una predicción comprensible sin necesidad de conocer los detalles técnicos del modelo, y recibir un mensaje claro cuando la fecha solicitada está fuera del rango válido de predicción.

---

## Descripción

El sistema ofrece una interfaz conversacional accesible desde el navegador donde el usuario puede escribir preguntas como *"¿Cuánto venderá Guayas en marzo de 2026?"* El sistema cubre datos históricos desde enero 2020 hasta septiembre 2025, y solo acepta solicitudes de predicción para fechas estrictamente posteriores a septiembre 2025 y dentro del horizonte máximo parametrizable (`MAX_PREDICTION_MONTHS`, por defecto 12 meses). Cuando la fecha solicitada no cumple estas condiciones, el chat muestra un mensaje amigable en español explicando el rango válido, sin exponer errores técnicos al usuario.

---

## Contexto del dominio temporal

| Parámetro | Valor |
|---|---|
| Primer registro del dataset | Enero 2020 |
| Último registro del dataset | Septiembre 2025 |
| Primer mes predecible | Octubre 2025 |
| Horizonte máximo | `MAX_PREDICTION_MONTHS` desde octubre 2025 (default: 12) |
| Fecha límite de predicción (default) | Septiembre 2026 |
| Variable de entorno | `MAX_PREDICTION_MONTHS=12` |

## Criterios de Aceptación

### Escenario 1: Pregunta válida con provincia, mes y año dentro del rango permitido

**Given** que el usuario está en la interfaz de chat
**And** escribe una pregunta que menciona una provincia válida, un mes y un año dentro del rango permitido (octubre 2025 – septiembre 2026 con configuración por defecto)
**When** presiona enviar o Enter
**Then** el sistema muestra un indicador visual de "procesando"
**And** en menos de 10 segundos aparece una respuesta en lenguaje natural con el valor de predicción
**And** la respuesta incluye provincia, mes, año y el valor predicho formateado en dólares (USD)
**And** se muestra un panel colapsable con datos técnicos: versión del modelo, score de confianza, provincia, mes, año y payload enviado al modelo

### Escenario 2: El usuario solicita predicción para una fecha histórica (anterior a octubre 2025)

**Given** que el usuario escribe una pregunta como *"¿Cuánto vendió Pichincha en junio de 2023?"*
**When** el agente valida la fecha extraída
**Then** el chat muestra un mensaje amigable como:
*"La fecha solicitada (junio 2023) pertenece al período histórico del dataset. El modelo predice únicamente desde octubre de 2025 en adelante. ¿Te gustaría consultar una fecha futura?"*
**And** no se realiza ninguna llamada al servicio de inferencia
**And** no se muestra ningún stack trace ni mensaje técnico al usuario

### Escenario 3: El usuario solicita predicción más allá del horizonte máximo configurado

**Given** que `MAX_PREDICTION_MONTHS=12` está configurado
**And** el usuario pregunta por una fecha posterior a septiembre 2026
**When** el agente valida la fecha
**Then** el chat muestra un mensaje como:
*"La fecha solicitada supera el horizonte máximo de predicción del modelo (12 meses). La última fecha válida es septiembre de 2026. ¿Te gustaría consultar dentro de ese rango?"*
**And** el mensaje calcula y muestra la fecha límite dinámicamente según el valor actual de `MAX_PREDICTION_MONTHS`
**And** no se realiza ninguna llamada al servicio de inferencia

### Escenario 4: El usuario no especifica el año en su pregunta

**Given** que el usuario escribe *"¿Cuánto venderá Guayas en marzo?"* sin mencionar el año
**When** el agente procesa la consulta
**Then** el agente infiere el año válido más próximo a partir de la fecha actual del sistema
**And** la respuesta del chat indica la interpretación realizada antes de mostrar la predicción: *"Interpreté tu consulta como marzo de 2026."*
**And** si el año inferido queda fuera del rango permitido, aplica el escenario 2 o 3 según corresponda

### Escenario 5: El historial de conversación se mantiene en sesión

**Given** que el usuario ha realizado al menos una consulta previa en la sesión actual
**When** realiza una nueva consulta
**Then** los mensajes anteriores permanecen visibles, incluyendo mensajes de error de validación temporal
**And** los nuevos mensajes se agregan al final del historial
**And** el scroll se posiciona automáticamente en el mensaje más reciente

### Escenario 6: Pregunta sin provincia identificable

**Given** que el usuario escribe una pregunta sin mencionar ninguna provincia del Ecuador
**When** el sistema procesa la pregunta
**Then** el agente responde solicitando la provincia de forma amigable
**And** no se muestra ningún error técnico ni stack trace

### Escenario 7: Accesibilidad del servicio

**Given** que el servicio web-ui está desplegado y corriendo
**When** el usuario accede a la URL raíz
**Then** la página carga en menos de 3 segundos
**And** el campo de input recibe el foco automáticamente
**And** el placeholder muestra el formato esperado: *"Ej: ¿Cuánto venderá Guayas en marzo de 2026?"*
**And** la interfaz funciona correctamente en Chrome, Firefox y Edge en sus versiones actuales
