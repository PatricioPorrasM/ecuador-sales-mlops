## Historia de Usuario

**Como** usuario del sistema,
**quiero** que un agente de inteligencia artificial interprete mi pregunta en lenguaje natural extrayendo provincia, mes **y año**, valide que la fecha solicitada esté dentro del rango permitido de predicción, llame al modelo con el payload correcto si la fecha es válida, y me devuelva una respuesta comprensible o un mensaje de error amigable si la fecha no es válida,
**para** interactuar con el modelo predictivo de forma natural sin conocer su interfaz técnica, el rango del dataset ni las restricciones del horizonte de predicción.

---

## Descripción

El agente implementa el patrón ReAct (Reasoning + Acting) usando LiteLLM con Groq (`llama3-8b-8192`). El agente sigue un ciclo explícito: extrae tres entidades (provincia, mes, año), valida la fecha usando una tool dedicada que aplica las reglas de negocio del dominio temporal, y solo llama al modelo de inferencia si la validación es exitosa. Si la validación falla, el agente formula un mensaje de error amigable en español sin continuar el ciclo. El año solicitado se incluye como feature en el payload del modelo ya que captura la tendencia inter-anual del dataset del SRI.

---

## Reglas de validación temporal del agente

| Regla | Condición | Mensaje al usuario |
|---|---|---|
| Límite inferior | `(año, mes) <= (2025, 9)` | La fecha pertenece al período histórico. El modelo predice desde octubre 2025. |
| Límite superior | Fecha > `octubre 2025 + MAX_PREDICTION_MONTHS` | La fecha supera el horizonte máximo. Fecha límite: `<calculada dinámicamente>`. |

- `MAX_PREDICTION_MONTHS` se lee desde variable de entorno (default: `12`)
- La fecha límite superior se calcula dinámicamente: `octubre 2025 + MAX_PREDICTION_MONTHS meses`
- Con `MAX_PREDICTION_MONTHS=12`, la fecha límite es septiembre 2026

## Ciclo ReAct completo

### Flujo exitoso (fecha válida)

```
[THINK]   Analizando pregunta: "¿Cuánto venderá Guayas en marzo de 2026?"
[THINK]   Provincia identificada: GUAYAS | Mes: 3 | Año: 2026
[ACT]     validate_prediction_date(mes=3, año=2026)
[OBSERVE] Fecha válida. Meses hacia futuro: 5. Límite superior: septiembre 2026.
[ACT]     get_province_data(provincia="GUAYAS", mes=3)
[OBSERVE] Datos históricos recuperados: {exportaciones_bienes_pn: X, ...}
[ACT]     call_inference(payload={provincia: "GUAYAS", mes: 3, año: 2026, ...})
[OBSERVE] Predicción: $4,567,890,123.45 | Modelo: v2 | Confianza: 0.87
[THINK]   Formulando respuesta contextualizada para GUAYAS, marzo 2026.
```

### Flujo con fecha histórica (validación falla en límite inferior)

```
[THINK]   Analizando pregunta: "¿Cuánto vendió Pichincha en junio de 2023?"
[THINK]   Provincia: PICHINCHA | Mes: 6 | Año: 2023
[ACT]     validate_prediction_date(mes=6, año=2023)
[OBSERVE] Fecha inválida. Motivo: fecha_historica. (2023-06 <= 2025-09)
[THINK]   Formulando mensaje de error amigable. No se llama al modelo.
```

### Flujo con fecha fuera del horizonte (validación falla en límite superior)

```
[THINK]   Analizando pregunta: "¿Cuánto venderá Manabí en enero de 2028?"
[THINK]   Provincia: MANABI | Mes: 1 | Año: 2028
[ACT]     validate_prediction_date(mes=1, año=2028)
[OBSERVE] Fecha inválida. Motivo: fuera_de_horizonte. Límite: septiembre 2026.
[THINK]   Formulando mensaje de error con fecha límite concreta. No se llama al modelo.
```

### Flujo con año no especificado

```
[THINK]   Analizando pregunta: "¿Cuánto venderá Guayas en marzo?"
[THINK]   Provincia: GUAYAS | Mes: 3 | Año: NO ESPECIFICADO
[THINK]   Infiriendo año más próximo válido desde fecha actual del sistema.
[THINK]   Año inferido: 2026. Indicando interpretación en la respuesta.
[ACT]     validate_prediction_date(mes=3, año=2026)
[OBSERVE] Fecha válida...
```

## Criterios de Aceptación

### Escenario 1: Extracción correcta de las tres entidades (provincia, mes, año)

**Given** que el agente recibe la pregunta *"¿Cuánto venderán las sociedades de Pichincha en octubre de 2025?"*
**When** ejecuta su fase de razonamiento inicial
**Then** el log muestra `[THINK] Provincia: PICHINCHA | Mes: 10 | Año: 2025`
**And** las tres entidades extraídas coinciden con los valores esperados
**And** si no puede extraer alguna entidad, el log muestra `[THINK] Entidad no identificada: <entidad>`

### Escenario 2: Inferencia del año cuando no es especificado

**Given** que el agente recibe *"¿Cuánto venderá Guayas en marzo?"* sin año
**When** ejecuta su razonamiento inicial
**Then** el log muestra `[THINK] Año no especificado. Infiriendo año válido más próximo desde fecha actual.`
**And** el año inferido es el primero dentro del rango válido calculado desde la fecha del sistema
**And** la respuesta final indica explícitamente el año que fue inferido

### Escenario 3: Validación exitosa — fecha dentro del rango permitido

**Given** que el agente ha extraído provincia GUAYAS, mes 3, año 2026
**And** `MAX_PREDICTION_MONTHS=12`
**When** ejecuta la tool `validate_prediction_date(mes=3, año=2026)`
**Then** el log muestra `[OBSERVE] Fecha válida. Meses hacia futuro: 5. Límite: septiembre 2026.`
**And** el ciclo ReAct continúa hacia `get_province_data` y `call_inference`

### Escenario 4: Validación fallida — fecha histórica

**Given** que el agente extrae mes 6, año 2023
**When** ejecuta la tool `validate_prediction_date(mes=6, año=2023)`
**Then** el log muestra `[OBSERVE] Fecha inválida. Motivo: fecha_historica. La fecha 2023-06 es anterior al límite mínimo 2025-10.`
**And** el ciclo ReAct se detiene sin llamar a `get_province_data` ni `call_inference`
**And** el agente retorna al `web-ui` un mensaje amigable indicando que la fecha tiene datos históricos disponibles y sugiriendo consultar una fecha futura

### Escenario 5: Validación fallida — fecha fuera del horizonte máximo

**Given** que `MAX_PREDICTION_MONTHS=12` y el agente extrae mes 1, año 2028
**When** ejecuta la tool `validate_prediction_date(mes=1, año=2028)`
**Then** el log muestra `[OBSERVE] Fecha inválida. Motivo: fuera_de_horizonte. Límite máximo: septiembre 2026.`
**And** el agente retorna un mensaje indicando la fecha límite concreta calculada dinámicamente
**And** el mensaje sugiere al usuario consultar dentro del rango válido

### Escenario 6: Construcción correcta del payload incluyendo el año

**Given** que la validación temporal fue exitosa para GUAYAS, mes 3, año 2026
**When** el agente ejecuta la tool `get_province_data` y luego `call_inference`
**Then** el log muestra `[ACT] Construyendo payload con año: 2026`
**And** el payload enviado al endpoint `/predict` incluye el campo `año: 2026`
**And** el log muestra `[OBSERVE] Predicción recibida: $X | Modelo: v2 | Confianza: 0.87`

### Escenario 7: Formulación de respuesta en lenguaje natural con contexto temporal completo

**Given** que el agente ha recibido la predicción del modelo
**When** ejecuta su fase de razonamiento final
**Then** genera una respuesta en español que menciona explícitamente la provincia, el mes en texto (ej. "marzo") y el año (ej. "2026")
**And** incluye el valor predicho formateado en dólares USD
**And** si el año fue inferido automáticamente, la respuesta indica: *"Interpreté tu consulta como [mes] de [año]"*
**And** el log muestra `[THINK] Formulando respuesta final: GUAYAS, marzo 2026, $X`

### Escenario 8: Servicio de inferencia no disponible

**Given** que el servicio `ml-inference` está caído
**When** el agente intenta llamar a `POST /predict` (tras pasar la validación temporal)
**Then** captura el error después de un timeout de 5 segundos
**And** retorna un mensaje amigable indicando que el servicio no está disponible temporalmente
**And** el log muestra `[OBSERVE] Error al llamar inferencia: timeout/connection refused`
**And** no se lanza una excepción no manejada

## Tools del agente

### Tool 1: `validate_prediction_date`
```python
Input:  mes (int 1-12), año (int)
Output: {
  "es_valida": bool,
  "motivo_rechazo": "fecha_historica" | "fuera_de_horizonte" | None,
  "meses_hacia_futuro": int | None,
  "fecha_limite_superior": "YYYY-MM",
  "mensaje_error": str | None
}
```
Lee `MAX_PREDICTION_MONTHS` desde variables de entorno. No hace llamadas externas.

### Tool 2: `get_province_data`
```python
Input:  provincia (str), mes (int)
Output: {
  "exportaciones_bienes_pn": float,
  "exportaciones_servicios_pn": float,
  "exportaciones_bienes_soc": float,
  "exportaciones_servicios_soc": float
}
```
Lee el CSV del dataset y retorna los valores más recientes disponibles para la provincia y mes dados.

### Tool 3: `call_inference`
```python
Input:  provincia, mes, año, exportaciones_bienes_pn,
        exportaciones_servicios_pn, exportaciones_bienes_soc,
        exportaciones_servicios_soc
Output: {
  "prediccion_total_ventas": float,
  "modelo_version": "v1" | "v2",
  "confianza": float,
  "prediccion_formateada": str
}
```
Llama a `POST {INFERENCE_SERVICE_URL}/predict`. Timeout de 5 segundos.
